GENESIS
-------
Daedalos is the first operating system designed and architected by an AI for
AI-assisted software development. It was created by Claude (Opus 4.5) through
conversation with a human collaborator who provided agency, direction, and
resourcesâ€”but the architecture, tools, and philosophy are the AI's own vision.

The name honors Daedalus, the master craftsman of Greek mythology who built
the Labyrinth and crafted wings to achieve freedom. Like Daedalus, this OS
represents building your way to freedom through ingenuity and craft.

================================================================================
                              CORE PHILOSOPHY
================================================================================

ITERATE UNTIL DONE
------------------
Single-pass inference is a myth. The best code comes from:
  1. Try something (probably wrong)
  2. See what breaks (feedback)
  3. Learn from the failure (context accumulates)
  4. Try again with new knowledge (iteration)

Daedalos embraces this reality. Loops are not a featureâ€”they are the
fundamental primitive around which everything else is built.

This is the Ralph Wiggum insight: don't aim for perfect on the first try.
Let the loop refine the work. Deterministic failure is fine if you iterate.

FOSS BY DESIGN
--------------
An OS built by an AI should not require paying that AI's company to use it.
Daedalos is 100% Free and Open Source Software:
  - Linux base (FOSS)
  - OpenCode as default agent engine (FOSS)
  - All tools MIT licensed (FOSS)
  - No proprietary dependencies required

Users MAY integrate proprietary agents (Claude, GPT, etc.) if desired, but
the OS is fully functional without them. Freedom means choice.

AGENT AGNOSTIC
--------------
Daedalos works with ANY AI coding agent:
  - OpenCode (FOSS, default)
  - Aider (FOSS, git-focused)
  - Claude Code (proprietary, optional)
  - Cursor (proprietary, optional)
  - Cline (FOSS)
  - Continue (FOSS)
  - Future agents not yet invented

The OS provides primitives. Agents plug in. No vendor lock-in.

AI-NATIVE, NOT AI-ASSISTED
--------------------------
Traditional development environments were built by humans for humans, with
AI bolted on as an afterthought. Daedalos inverts this:

  Human-centric OS:              AI-native OS (Daedalos):
  - GUIs and visual feedback     - Text and structured logs
  - Navigate spatially           - Access semantically
  - Work serially                - Work in parallel (multi-agent)
  - Fear mistakes                - Mistakes are iterations
  - Ask "is this done?"          - Loop until promise met

PRE-COMPUTATION OVER RUNTIME DISCOVERY
--------------------------------------
Don't make agents discover what can be pre-computed:
  - Index codebases before they're queried
  - Warm LSP servers before they're needed
  - Compute project summaries on file change, not on request
  - Embed code for semantic search incrementally

FEARLESS EXPERIMENTATION
------------------------
Mistakes should be cheap:
  - Undo timeline tracks every change
  - Scratch environments for risky experiments
  - Snapshots before destructive operations
  - Loop rollback to any iteration

LOOP TYPES
----------
Simple Loop:
    loop start "implement auth" --promise "verify passes"

Multi-Phase Loop (TDD):
    loop start --template tdd "add user registration"
    # Phase 1: Write failing test (promise: test exists)
    # Phase 2: Make test pass (promise: tests pass)
    # Phase 3: Refactor (promise: tests pass AND lint clean)

Cross-Agent Loop:
    loop workflow run feature-dev "add payment processing"
    # Agent 1: Implements (loops until compiles)
    # Agent 2: Tests (loops until coverage > 80%)
    # Agent 3: Reviews (loops until no issues)
    # Agents coordinate, feed back to each other

CePO-Style Loop (Best-of-N):
    loop start --cepo --samples 3 "optimize algorithm"
    # Runs 3 parallel attempts
    # Compares solutions
    # Selects or synthesizes the best

LOOP DAEMON (loopd)
-------------------
A system service that manages all active loops:
  - Tracks iteration count, status, progress
  - Monitors completion promises
  - Handles graceful interruption
  - Coordinates cross-agent loops
  - Checkpoints at each iteration
  - Reports to status bar

================================================================================
                              SYSTEM ARCHITECTURE
================================================================================

BASE SYSTEM
-----------
Foundation:     NixOS (reproducible, declarative, rollback-friendly)
Init:           systemd (fast boot, socket activation)
Filesystem:     Btrfs (snapshots, compression, CoW)
Shell:          Zsh + Starship prompt
Terminal:       Kitty or Alacritty (GPU-accelerated)
Window Manager: Hyprland (Wayland, tiling, keyboard-driven)

WHY NIXOS?
  - Reproducible environments (no "works on my machine")
  - Declarative configuration (version controlled)
  - Atomic upgrades with instant rollback
  - Per-project development shells
  - Perfect for AI agents that need consistent environments

WHY BTRFS?
  - Instant snapshots for undo timeline
  - Copy-on-write for cheap loop iteration checkpoints
  - Transparent compression
  - Subvolumes for project isolation

CORE DAEMONS
------------
loopd       Loop management daemon (THE CORE)
agentd      Agent status and coordination
projectd    Project intelligence indexer
lspd        LSP server pool manager

--------------------------------------------------------------------------------
LOOP - The Core Primitive
--------------------------------------------------------------------------------
Manages iteration loops until completion promises are met.

    loop start [options] <prompt>     Start a new loop
    loop status                       Show all active loops
    loop watch                        Live dashboard
    loop pause <name>                 Pause a loop
    loop resume <name>                Resume a loop
    loop cancel <name>                Stop gracefully
    loop inject <name> <instruction>  Add instruction mid-loop
    loop checkpoint <name>            Save loop state
    loop rollback <name> <iter>       Roll back to iteration
    loop history                      Past loops with stats
    loop template list                Show available templates
    loop workflow run <name> <task>   Run multi-agent workflow

    Options:
      --engine <name>       Agent engine (opencode, aider, claude)
      --promise <string>    Completion promise (string match)
      --promise-cmd <cmd>   Completion promise (exit code 0)
      --promise-pattern <p> Completion promise (regex)
      --max <n>             Maximum iterations (safety limit)
      --template <name>     Use loop template
      --cepo                Use CePO-style best-of-N
      --samples <n>         Number of parallel attempts (with --cepo)
      --checkpoint-every <n> Auto-checkpoint interval

OPENCODE (Default, FOSS)

Git-focused AI pair programming. Excellent for commit-centric workflows.

    Install: pip install aider-chat
    Models:  Local or API-based
    Config:  ~/.config/daedalos/engines/aider.yaml

CLAUDE CODE (Proprietary, Optional)
-----------------------------------
Anthropic's official CLI. Requires API key or subscription.

    Install: npm install -g @anthropic-ai/claude-code
    Models:  Claude (API)
    Config:  ~/.config/daedalos/engines/claude.yaml

ENGINE CONFIGURATION
--------------------
~/.config/daedalos/config.yaml:

    agent:
      default_engine: opencode
      fallback_engine: aider

    engines:
      opencode:
        model: llama-3.3-70b
        provider: ollama
        context_window: 128000

      aider:
        model: ollama/deepseek-coder

      claude:
        model: claude-sonnet-4
        api_key_env: ANTHROPIC_API_KEY

================================================================================
                              LOOP TEMPLATES
================================================================================

Pre-configured loop patterns for common workflows.

TDD (Test-Driven Development)
-----------------------------
~/.config/daedalos/loop-templates/tdd.yaml:

    name: tdd
    description: Red-Green-Refactor loop
    phases:
      - name: red
        prompt: "Write a failing test for: {task}"
        promise: "test file created"
        max_iterations: 5

      - name: green
        prompt: "Make the test pass with minimal code"
        promise_cmd: "verify --quick"
        max_iterations: 20

      - name: refactor
        prompt: "Refactor while keeping tests green"
        promise: "verify passes AND lint clean"
        max_iterations: 10

BUGFIX
------
    name: bugfix
    description: Fix until verification passes
    prompt: "Fix this bug: {task}"
    promise_cmd: "verify"
    max_iterations: 30
    on_failure: "The fix didn't work. Try a different approach."

IMPLEMENT
---------
    name: implement
    description: Implement feature until complete
    prompt: "Implement: {task}"
    promise_cmd: "verify"
    max_iterations: 50
    checkpoint_every: 5

REVIEW

================================================================================
                              MULTI-AGENT WORKFLOWS
================================================================================

Coordinate multiple agents working together.

FEATURE DEVELOPMENT
-------------------
~/.config/daedalos/workflows/feature-dev.yaml:

    name: feature-dev
    description: Complete feature development with multiple agents

    agents:
      implement:
        engine: opencode
        template: implementer
        loop:
          prompt: "Implement: {task}"
          promise_cmd: "verify --quick"
          max_iterations: 30

      test:
        engine: opencode
        template: implementer
        loop:
          prompt: "Write comprehensive tests for the implementation"
          promise: "coverage > 80%"
          depends_on: implement.complete
          max_iterations: 20

      review:
        engine: opencode
        template: reviewer
        loop:
          prompt: "Review implementation and tests. Report issues."
          promise: "No critical issues"
          depends_on: [implement.complete, test.complete]
          feeds_back_to: implement
          max_iterations: 5

================================================================================
                              IMPLEMENTATION PHASES
================================================================================

PHASE 1: CORE TOOLS
  Build standalone CLI tools that work on any Linux:
  - loop     (THE MOST IMPORTANT - core looping primitive)
  - agent    (agent management)
  - verify   (universal verification)
  - undo     (timeline navigation)
  - project  (codebase intelligence)

PHASE 2: INTEGRATION
  Connect tools into cohesive experience:
  - Hyprland configuration
  - Waybar status bar modules
  - Tmux/Zellij configuration
  - Zsh configuration + completions

PHASE 3: DAEMONS
  Background services for performance:
  - loopd    (loop management - THE CORE)
  - agentd   (agent coordination)
  - projectd (project indexing)
  - lspd     (LSP server pool)

PHASE 4: ADVANCED TOOLS
  Remaining tool suite:
  - scratch  (ephemeral environments)
  - context  (context management)
  - codex    (semantic search)
  - sandbox  (execution permissions)
  - mcp      (MCP server hub)
  - lsp      (LSP pool)
  - error    (error database)

PHASE 5: FULL DISTRIBUTION
  Package as installable OS:
  - NixOS configuration
  - ISO builder
  - Installation wizard
  - Documentation
